anatomy of an options print

One options execution starts as an OPRA-style trade message. Islandflow turns it into an enriched print, keeps or rejects it for the signal tape, clusters it into flow packets, projects canonical smart-flow hypotheses, and emits smart-flow alerts only when scored evidence survives the guards.

The print record

The canonical OptionPrint is not just a trade. It is a trade plus execution context, quote context, derived contract metadata, and signal gating state. The raw fields give provenance; the enriched fields let later stages reason without rejoining every quote on the client.

Pipeline topology

The live pipeline is event-sourced. Adapters publish raw prints and quotes, ingest writes enriched option prints, compute derives packets and alerts, API hydrates hot caches and history, and the web client consumes scoped live snapshots.

Raw streams

option_prints, option_nbbo, equity_prints, and equity_quotes keep source events with raw-stream retention.

Derived streams

option_signal_prints, flow_packets, smart_flow_projections, and smart_flow_alerts keep compute products with derived-stream retention.

Print enrichment

ingest-options keeps short quote histories by contract and underlying. On each print it selects the latest option NBBO and equity quote at or before the trade timestamp, derives metadata, records execution context, and only then evaluates the signal policy.

Contract parse

The parser accepts dashed contracts and OCC-style symbols. It extracts root, expiry, strike, and right, then maps C to call and P to put.

Quote join

Option and equity quote histories keep up to 64 items per key. The selected quote is the latest item at or before the print timestamp.

Signal inputs

Notional is price * size * 100. Quote side, ETF status, conditions, size, and notional drive the downstream keep or reject reasons.

NBBO side classification

The tolerance is max(0.01, spread * 0.05). Trades outside the ask or bid by more than that tolerance become AA or BB.

Signal gate

The options tape defaults to signal view. The gate is deterministic and explainable: every reject or keep path emits a reason string, so the UI can show why a print made it through.

Reject rules

  • reject:min-notional when notional is below the active profile minimum.
  • reject:etf-min-notional for ETF underlyings below the ETF premium floor.
  • reject:bid-side-min-notional for bid-side flow below the bid-side floor.
  • reject:mid-min-notional for mid prints without sweep or ISO context.
  • reject:missing-nbbo-min-notional when quote context is missing or stale below its higher floor.
  • reject:no-signal-rule when the print clears floors but lacks any positive keep rule.

Keep rules

  • keep:auto-large for very large premium, currently 100,000 in smart-flow mode.
  • keep:ask-side for ask-side or above-ask prints above the profile minimum.
  • keep:sweep-or-iso for sweep or ISO prints above the sweep floor.
  • keep:large-contract-count for large contract count with enough premium.
  • mode:all bypasses signal filtering when explicitly selected.
Profile Intent Important thresholds
smart-flow Strict default for OPRA signal prints. 10k minimum, 50k ETF floor, 25k bid-side floor, 20k mid floor, 50k missing or stale quote floor, 100k auto keep.
balanced Lower-friction mode that relaxes floors without going raw. 5k minimum, 25k ETF floor, 15k bid-side floor, 12.5k mid floor, 25k missing or stale quote floor, 75k auto keep.

Flow packets

Compute consumes the signal-print stream and groups nearby prints by contract. The default cluster window is 500 ms. When a contract has been quiet long enough, the cluster flushes as a FlowPacket, writes to ClickHouse, publishes to JetStream, and immediately enters the alert path.

Single-contract packets

The packet id is built from contract id, start timestamp, and end timestamp. Its features include first and last price, window bounds, member count, premium, notional, NBBO counts, quote context, underlying move, IV shock, special-print count, and rolling baselines.

Structure packets

Recent legs by root and expiry are summarized into straddles, strangles, verticals, ladders, and rolls. Structure packets use pseudo contract ids and include aggregate NBBO placement, same-size symmetry, roll strikes, and roll expiries.

Suppression happens before alerting. Special, complex, cross, auction, opening, closing, stale quote, missing quote, and inside-market contexts can force the event to abstain. An abstained projection produces no smart-flow alert.

Smart-flow scoring

Smart-flow is the canonical vocabulary for evidence-backed hypotheses. The current API exposes persisted smart-flow explainability projections. The compute package contains the candidate, cluster, hypothesis, and replay scoring machinery used for runtime emission and evaluation.

Canonical object chain

  • FlowObservationRef points to packet, print, quote, news, event, or label evidence.
  • FlowEvidenceFact gives a labeled, traceable fact with refs.
  • FlowCandidate wraps one packet with feature vector, quality, and eligibility.
  • FlowEvidenceCluster groups candidates by underlying and time window.
  • FlowHypothesisEvent selects a scored hypothesis or abstains.
  • SmartFlowInsight turns the event into UI-facing summary language.

Evidence quality

Quality combines quote coverage, stale ratio, and completeness. Grades are strong, usable, thin, and poor. Candidate eligibility can accept, reject, or down-weight a packet before it becomes a cluster.

Hypotheses

The scorer ranks directional accumulation, retail attention flow, event positioning, volatility supply, structure arbitrage, hedge rebalance, and unclear.

Penalties

Stale quotes, wide quotes, inside-market context, complex prints, low premium, weak aggression, missing context, conflicting direction, and structure context reduce confidence.

Abstention

The projection abstains when evidence is insufficient, confidence is below policy, the margin is too small, penalties are too large, or the best answer is unclear.

How an alert fires

Alerts are smart-flow hypothesis events. They are not emitted from every option print, and they are not emitted from an abstained projection. The runtime path is compact: packet flush, smart-flow projection, smart-flow alert derivation, storage, publish, fanout.

Packet flushes

flushCluster builds a FlowPacket, stores it, publishes it, and projects canonical smart-flow evidence.

Smart-flow projection is built

The native smart-flow runtime builds evidence clusters, scores hypotheses, and records penalties and abstention. If the projection abstains, the alert path ends.

Smart-flow alert is projected

A non-abstained projection becomes a SmartFlowAlertEvent with the same hypothesis, confidence, evidence quality, and evidence refs.

Trigger metadata is calculated

The alert records trigger kind, policy confidence, evidence quality, and optional throttle/threshold reasons without a legacy score or severity field.

The alert is persisted and published

The alert stores its canonical payload in ClickHouse, publishes on the smart-flow-alerts stream, and fans out through REST, history, replay, and live channels.

Storage and API

ClickHouse is the historical source of truth. Redis is a hot-head cache for live snapshots and cursors. The API serves current head, older history, replay-after windows, direct support lookups, and websocket feeds.

Columnar print storage

option_prints stores execution context in first-class columns and uses a bloom-filter index on trace_id for bounded evidence lookup.

JSON feature storage

flow_packets, smart_flow_projections, and smart_flow_alerts store feature and evidence payloads as JSON alongside ordered source cursors.

Web surfaces

The web app does not treat prints as isolated rows. The OPRA intake route keeps the tape, packet focus, NBBO context, smart-flow evidence, and alert evidence connected so users can move from a row to the supporting events.

OPRA intake

/options subscribes to options, NBBO, and flow packets. The options tape supports signal/raw view, side/type/security/premium filters, contract focus, and packet-member focus.

Flow packet tape

Packet rows expose total premium, notional, member count, quote state, aggression ratios, structure labels, and an inspect action that focuses member prints.

Alert detail

Alert drawers hydrate persisted context by trace id, then show the canonical smart-flow alert, backing projection evidence, packet, member prints, missing refs, and focus actions for packet, contract, and equity.
