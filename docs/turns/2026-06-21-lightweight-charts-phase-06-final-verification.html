Browser QA

  • Started only bun run dev:web. The web dev server ran on http://localhost:3000 and reported NEXT_PUBLIC_API_URL=https://api.flow.deltaisland.io; no local infra, full stack, or backend services were started.
  • Desktop 1440x960: dashboard rendered as the flat Market Command section; chart toolbar, lower pane, decision levels, and adjacent tables were aligned without visible overlap.
  • Desktop settings menu: chart settings dialog measured 380x620 inside the viewport, with price, lower chart, timeframe, and display sections visible and unsupported controls disabled.
  • Timeframe control: native combobox exposes 1m, 5m, and 15m as enabled options and 30m unavailable/1h unavailable as disabled options. The OS-native dropdown surface is not exposed as DOM or captured by the browser screenshot layer.
  • Desktop hover readout: chart-area click activated crosshair snapshots; left and right plot-edge probes produced bounded readouts with OHLC, volume, trades, flow context, and lower-pane rows. The dashboard chart is under the compact tooltip threshold, so the readout uses the bounded full-width treatment.
  • Mobile 390x844: top controls stacked cleanly; chart controls wrapped into stable rows; chart section measured 374px wide; settings dialog measured 370x560 and stayed inside the viewport.
  • Mobile hover readout: left and right probes produced a 358px wide tooltip inside the viewport with no clipping.
  • Empty/loading/error state pass: a practical nonmatching ticker probe found a stale-data defect instead of the expected empty/loading state. Error state was not forced because doing so would require breaking backend/API configuration during closeout.
