Dashboard V2 Live API Edge Recovery

Fixed the native dashboard failure where /market-command/tickers reached the web app and returned an HTML 404, while dashboard-shaped live subscriptions could starve the API event loop and make the backend look unavailable.

Branchdashboard-v2
Beadsislandflow-hzlf
Runtimenative user service + NPM
ScopeAPI live state, edge matcher, probes

What Failed

The screenshot had two independent symptoms. The HTTP 404: received HTML response instead of JSON warning came from NPM not forwarding /market-command/* to the API. Separately, dashboard reconnects pushed islandflow-api into high CPU and health latency because live durable-row composition and websocket send bursts were too synchronous under the root dashboard subscription set.

Implementation

  • Added market-command to the native/docker same-origin API matcher and to the public route probe.
  • Replaced boolean env coercion with explicit string parsing so API_CONSUMER_RESET=false stays false.
  • Changed durable-row live event composition to resolve smart-flow support only for event-matched option rows.
  • Made Redis live-cache flushing single-flight and tolerant of a Redis client closing during shutdown.
  • Yielded during large subscribe snapshots and live fanout bursts so health/REST requests get event-loop time.
