Summary

Implemented the smart-flow phase 1B foundation by adding canonical shared contracts for observations, candidates, evidence clusters, hypotheses, confidence vectors, abstention, baselines, and user-facing insight projections.

Changes Made

  • Added packages/types/src/smart-flow.ts with versioned Zod schemas and inferred TypeScript types.
  • Exported the smart-flow contract module from packages/types/src/index.ts.
  • Marked SmartMoneyEventSchema as the current compatibility feed shape.
  • Added SmartMoneyInsightSchema and SmartMoneyInsight as deprecated aliases for compatibility-only projection language.
  • Added schema tests for versioning, evidence clustering, confidence versus abstention, and legacy projection.
  • Added phase migration notes for future implementation phases.

Context

docs/implementation/README.md identifies phase 1B as islandflow-zxh.1, under the smart money / smart flow epic. The phase can run in parallel with synthetic phase 1A and exists to define careful vocabulary before evidence extraction or scoring changes happen.

The implementation stayed inside the phase scope: contracts, vocabulary, version fields, compatibility aliases, migration notes, and tests.

Important Implementation Details

  • SMART_FLOW_CONTRACT_VERSION, SMART_FLOW_POLICY_VERSION, and SMART_FLOW_MODEL_VERSION establish explicit evolution points.
  • FlowCandidate and FlowEvidenceCluster separate raw observation references from derived evidence facts.
  • FlowHypothesisEvent separates hypothesis type, score vector, alternatives, confidence vector, and abstention reasons.
  • SmartFlowInsight is the user-facing projection contract and does not claim hidden participant identity as fact.
  • The legacy projection helper maps existing smart-money profile IDs into cautious smart-flow hypothesis types without changing existing feeds.

+# Smart-Flow Phase 01 Migration Notes
+
+These notes accompany Beads issue islandflow-zxh.1 and the contracts in
+packages/types/src/smart-flow.ts.
+
+## Canonical language
+
+The canonical contract family is smart-flow, not smart-money.
+
+- Observations are direct references to market, news, calendar, or fixture inputs.
+- Evidence facts are normalized statements derived from observations.
+- Evidence clusters group candidates, facts, baselines, and evidence quality.
+- Hypotheses are policy/model outputs over evidence, not facts about hidden participants.
+- Confidence vectors describe policy confidence, evidence quality, margin, conviction, and calibration status separately.
+- Abstention is a first-class output with reasons and source reasons.
+- Insights are user-facing projections from hypotheses.
+
+## Compatibility posture
+
+Existing SmartMoneyEvent feeds, storage tables, and UI paths stay compatibility surfaces for phase 01.
+
+SmartMoneyInsight remains as a deprecated alias for SmartFlowInsight so existing API/UI naming can migrate without implying a hidden-participant fact.
+

Expected Impact for End-Users

There is no direct UI or API behavior change in this phase. The user-visible effect comes later: future smart-flow outputs can explain what was observed, what evidence was derived, what hypothesis was formed, how confident the policy is, and why the system abstained.
