Smart-flow Chart Markers

Moved terminal chart and focus marker rendering onto smart-flow explainability projections, while retaining legacy smart-money rows only as compatibility fallback.

Branchlavender/smart-flow-chart-markers
Beadsislandflow-zxh.5.3
ScopePhase 05 API/UI explainability follow-up

Summary

The chart and Focus pane now prefer canonical smart-flow explainability projections for flow markers. Raw legacy smart-money events remain available only when no smart-flow projection is available for the visible marker source.

Changes Made

  • Added a chart marker source helper that filters, sorts, caps, and selects smart-flow projections before legacy smart-money fallback events.
  • Updated candle chart marker rendering so smart-flow markers use hypothesis/abstention text and open the smart-flow explainability drawer.
  • Updated the Focus pane to list smart-flow hypotheses with direction, evidence quality, compatibility projection status, and timestamp.
  • Added focused tests for smart-flow marker precedence, legacy fallback behavior, and stable source-time ordering.

Context

This is a follow-up to phase 05b. PRs #40 and #41 already introduced smart-flow API, websocket, and UI explainability surfaces. This task closes the remaining chart/focus rendering path that still centered raw smart-money events.

The implementation stays within the phase boundary: no scoring policy changes, no calibration, no synthetic foundations, and no participant-identity claims.

Important Implementation Details

  • getChartFlowMarkerItems returns smart-flow projection marker items when any are available in the visible range.
  • Legacy SmartMoneyEvent marker rows are now modeled as smart-money-fallback items and only fill gaps when the canonical projection list has no in-range items.
  • Smart-flow chart markers open SmartFlowDrawer through openFromSmartFlowProjection; legacy fallback markers still open the compatibility smart-money drawer.
  • Focus rows use cautious smart-flow language: hypothesis label, evidence quality, abstained direction, and compatibility projection status when relevant.

export type ChartFlowMarkerItem =
  | { kind: "smart-flow"; projection: SmartFlowExplainabilityProjection }
  | { kind: "smart-money-fallback"; event: SmartMoneyEvent };


const sortBySourceTime = <T extends { source_ts: number; seq: number }>(items: readonly T[]): T[] =>
  [...items].sort((a, b) => {
    const delta = a.source_ts - b.source_ts;
    if (delta !== 0) {
      return delta;
    }
    return a.seq - b.seq;
  });


export const getChartFlowMarkerItems = (
  smartFlowProjections: readonly SmartFlowExplainabilityProjection[],
  legacySmartMoneyEvents: readonly SmartMoneyEvent[],
  visibleRangeMs: { from: number; to: number } | null,
  maxSmartFlowMarkers = 220,
  maxLegacySmartMoneyMarkers = 220
): ChartFlowMarkerItem[] => {
  if (!visibleRangeMs) {
    return [];
  }


  const inRangeSmartFlow = sortBySourceTime(
    smartFlowProjections.filter(
      (projection) =>
        projection.source_ts >= visibleRangeMs.from && projection.source_ts <= visibleRangeMs.to
    )
  );


  if (inRangeSmartFlow.length > 0) {
    const cappedSmartFlow =
      inRangeSmartFlow.length > maxSmartFlowMarkers
        ? inRangeSmartFlow.slice(inRangeSmartFlow.length - maxSmartFlowMarkers)
        : inRangeSmartFlow;
    return cappedSmartFlow.map(
      (projection): ChartFlowMarkerItem => ({ kind: "smart-flow", projection })
    );
  }


  const inRangeLegacy = sortBySourceTime(
    legacySmartMoneyEvents.filter(
      (event) => event.source_ts >= visibleRangeMs.from && event.source_ts <= visibleRangeMs.to
    )
  );
  const cappedLegacy =
    inRangeLegacy.length > maxLegacySmartMoneyMarkers
      ? inRangeLegacy.slice(inRangeLegacy.length - maxLegacySmartMoneyMarkers)
      : inRangeLegacy;
  return cappedLegacy.map((event): ChartFlowMarkerItem => ({ kind: "smart-money-fallback", event }));
};
