Native Alert Evidence, Time Zone, and Chart Stability Repair

Fixed the native dashboard path where alert details repeatedly tried to hydrate quote context as print evidence, stale synthetic alert rows leaked into the live dashboard, module clocks disagreed, and live candle updates replaced the chart history window.

What Failed

Alert evidence refs mixed packet refs, option-print refs, and option-nbbo: quote-context refs. The drawer treated quote context like fetchable print evidence, so it kept retrying refs that were never option prints. Storage lookup also allowed multiple rows per trace id, which could surface unrelated symbols in evidence previews.

The deployed native instance was still carrying synthetic live alert rows in Redis. The REST route was cleaned first, but the live dashboard still hydrated live:smart-flow-alerts until the live manager also learned the native-mode synthetic filter.

Time display was split between hard-coded Eastern labels, server-rendered row strings, and browser-local chart labels. The chart instability came from live candle arrays replacing the REST bootstrap history, so state updates could collapse the visible window down to the hot head and make earlier bars appear to flicker.

Implementation

  • Separated alert evidence into packet refs, print refs, and quote-context refs; option-nbbo: now renders as context instead of a failing print lookup.
  • Changed option-print trace lookup to return the newest single row per trace id with LIMIT 1 BY trace_id.
  • Filtered synthetic smart-flow alerts whenever hosted adapters are not synthetic, covering REST history, live Redis hydration, live ingest, durable rows, and direct alert websockets.
  • Moved UI time formatting through one browser-aware helper: user timezone by default, falling back to America/New_York.
  • Made chart live updates merge into the REST candle bootstrap window instead of replacing it, with a stable 300-candle cap and ticker/interval filtering.
  • Removed duplicate Alpaca websocket auth sends from the options and equities adapters.
