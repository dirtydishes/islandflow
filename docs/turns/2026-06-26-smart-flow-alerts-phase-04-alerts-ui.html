Islandflow implementation turn

Smart-flow alerts phase 04 migrated the alerts UI to canonical hypothesis alerts.

Beads islandflow-ghce.4 Branch lavender/smart-flow-alerts-04-alerts-ui 2026-06-26

Summary

Moved the web alerts module off legacy AlertEvent rows and onto canonical SmartFlowAlertEvent hypothesis alerts. The rows now expose time, symbol, hypothesis, direction, and confidence/evidence; legacy score, severity, and classifier-hit presentation is gone from the canonical alert UI. The detail drawer now starts with fast triage and keeps packet, contract, and equity focus handoffs owned by the alerts module.

Implementation

Canonical alert source

AlertsModule, its source adapter, filters, row keys, and history loading now consume SmartFlowAlertEvent and /history/smart-flow-alerts.

Triage-first detail

The drawer leads with symbol, hypothesis, direction, trigger, confidence/evidence quality, and primary packet or option refs before alternatives, why-not context, penalties, and versions.

Shared tinting

Alert rows call the shared smart-flow tint module and map its metadata into row classes and CSS variables instead of recomputing tint locally.

Terminal cutover

Terminal alert transport, live manifest, replay paths, and command surfaces now use smart-flow-alerts while keeping transport/state thin.

Evidence ownership

The alerts feature owns packet and option-print evidence hydration through direct canonical refs, without reading legacy alert context for normal rendering.

  • Canonical alerts UI: apps/web/features/alerts/AlertsModule.tsx, columns.tsx, format.ts, source.ts, evidence.ts, types.ts, tinting.ts, and exports.

  • Styles and tests: apps/web/app/globals.css, apps/web/features/alerts/alerts-module.test.ts, and apps/web/app/terminal.test.ts.
