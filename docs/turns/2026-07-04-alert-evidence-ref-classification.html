Alert Evidence Ref Classification

Stopped the alerts drawer and terminal evidence helpers from treating contextual evidence refs as option-print trace ids. This removes the false persisted-context miss path for refs such as equity-quote:*.

Root Cause

Smart-flow alert evidence refs are mixed observation references. A single evidence_refs list can include flow packets, option prints, option NBBO, equity quotes, labels, and other external context.

The drawer had only a narrow split: flowpacket:*, option-nbbo:*, and everything else. That made refs like equity-quote:SPY:1000 look like option-print trace ids, so the client sent them to /option-prints/by-trace. Those refs were not option-print rows, so the lookup correctly had no persisted option-print context, but the UI interpreted the miss as drawer context instability.

Implementation

  • Added a shared alert ref classifier for flow packets, known context refs, and option-print refs.
  • Classified option-nbbo:, equity-quote:, equity-print:, news-story:, event-calendar:, synthetic-label:, and external-context: as context refs.
  • Updated alert primary-option selection and terminal smart-flow print grouping to use the same classifier.
  • Kept unresolved, non-context refs eligible for option-print hydration so existing provider trace ids still work.

Durable Contract Fix

The stronger fix is server-side typed evidence: either a typed alert evidence contract or an alert evidence resolver endpoint that returns resolved items by kind. I filed islandflow-3q0e for that work. This patch removes the immediate broken routing path while keeping the deeper contract cleanup tracked.
