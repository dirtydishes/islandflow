Smart Flow Score Vectors

Implemented the smart-flow phase 03a child issue: evidence clusters now produce deterministic, versioned hypothesis score vectors with direction, confidence, conviction, penalties, and preserved negative evidence.

Branchlavender/smart-flow-score-vectors
ScopeScore vectors only, no API or UI projection
ValidationTypecheck and full tests passed

Summary

Added the first canonical scoring step after evidence clustering. The new scorer ranks supported smart-flow hypothesis types from FlowEvidenceCluster inputs and returns typed FlowHypothesisScoreVector records. It does not emit final hypothesis events, abstention outputs, API fields, or UI projections, which remain in the next split issue.

Changes Made

  • Extended smart-flow contracts with score policy/model version constants, typed penalty kinds, FlowScorePenalty, and self-describing score vectors.
  • Added services/compute/src/smart-flow-scoring.ts, a pure rule-based scorer over FlowEvidenceCluster.
  • Enriched evidence candidates and cluster summaries with traceable option side, buy/sell aggression, net directional bias, execution IV, underlying movement, event timing, and structure facts.
  • Preserved execution_iv on emitted compute flow packets so volatility-supply scoring can use observed IV context later.
  • Added scoring tests for clean directional flow, negative evidence penalties, and neutral structure-led scoring.

Context

docs/implementation/README.md places smart-flow hypothesis scoring after synthetic scenarios and labels. The selected ready child issue, islandflow-zxh.3.1, is the phase 03a split for hypothesis score vectors only. The parent phase also includes abstention and insight projection, but those are explicitly left for islandflow-zxh.3.2.

The implementation follows the phase document's caution: confidence is policy confidence in observed evidence, not a hidden participant identity claim.

Important Implementation Details

  • scoreFlowEvidenceCluster returns ranked vectors for directional accumulation, retail attention flow, event positioning, volatility supply, structure arbitrage, hedge rebalance, and unclear.
  • Each vector carries schema_version, policy_version, model_version, hypothesis_type, direction, evidence_strength, fit_score, penalty_score, penalties, and a confidence vector.
  • Penalty records include machine-readable kinds, scores, reasons, feature keys, and evidence refs. Wide quotes, stale context, inside-market context, low premium, weak aggression, conflicting direction, and missing context are not flattened away.
  • Legacy smart-money compatibility events now populate the stricter score-vector shape without becoming the canonical scoring model.

+export const scoreFlowEvidenceCluster = (
+  cluster: FlowEvidenceCluster,
+  config: FlowHypothesisScoringConfig = {}
+): FlowHypothesisScoreVector[] => {
+  const resolvedConfig = { ...DEFAULT_CONFIG, ...config };
+  const sortedDrafts = sortDrafts(scoreHypotheses(buildContext(cluster)));
+  const maxVectors = Math.max(1, Math.round(resolvedConfig.maxVectors));
+
+  return sortedDrafts.slice(0, maxVectors).map((draft, index, drafts) => {
+    const nextConfidence = drafts[index + 1]?.policyConfidence ?? 0;
+    return toScoreVector(draft, resolvedConfig, draft.policyConfidence - nextConfidence);
+  });
+};

Issues, Limitations, and Mitigations

  • The scoring policy is rule-based and deterministic. Historical calibration remains explicitly out of scope for this MVP phase.
  • The scorer returns vectors but does not emit final FlowHypothesisEvent records. That keeps this PR inside islandflow-zxh.3.1 and leaves abstention/projection to islandflow-zxh.3.2.
  • The Diffs SSR output was not embedded because its generated payload was disproportionate for this small explanatory record. Styled unified fallback snippets preserve actual file and line-level changes.
