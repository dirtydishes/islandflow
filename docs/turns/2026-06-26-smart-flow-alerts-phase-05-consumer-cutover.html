Smart-Flow Alerts Phase 05 Consumer Cutover

Normal product consumers now use canonical smart-flow and smart-flow-alerts surfaces. Legacy smart-money, classifier-hit, and legacy alert feeds are removed from public runtime UI paths and explicit transition responses cover old API routes.

Beads islandflow-ghce.5 Branch lavender/smart-flow-alerts-05-consumer-cutover Spec docs/implementation/smart-flow-alerts/05-consumer-cutover.md

Implementation

Route and live manifest cutover

Terminal route features subscribe to smart-flow and smart-flow-alerts. Normal UI live buffers, transport types, history paths, and route manifests no longer include smart-money, classifier-hits, or legacy alerts.

Terminal and durable consumers

Legacy classifier and smart-money drawers, row decoration, hydration, chart marker callbacks, and option support merges were removed. Durable option and alert rows now compose from canonical smart-flow projections and smart-flow alert events.

Canonical chart and tape behavior

Chart markers, lower-pane smart-direction bars, hover rows, and options tape tinting no longer silently fall back to legacy smart-money or classifier-hit data. Missing canonical data now appears as empty states or diagnostics.

Transition API responses

Old derived HTTP and websocket routes return explicit 410 deprecation payloads with canonical replacements. Live session subscriptions to legacy derived channels receive an error naming the replacement channel.

Changed Files

  • Terminal cutover: apps/web/features/terminal/* route features, state, live buffers, transport, chart adapter, chart components, drawers, hydration scheduler, filters, and tests.
  • Consumer modules: durable tape row view models, market-chart lower-pane and hover transforms, options tape types/tinting, and related tests.
  • API cutover: services/api/src/index.ts, durable-rows.ts, live.ts, options-support.ts, plus new legacy-derived-routes.ts helper and tests.
  • Turn documentation: this closeout file in docs/turns.

Risks

  • Browser QA note: the managed API on port 4000 is production-like and was not restarted for this branch. To avoid competing for NATS durable consumers, browser QA used a temporary mock API that accepts canonical live subscriptions and rejects legacy ones. Branch API behavior is covered by services/api/tests.
  • Legacy storage helpers, ClickHouse tables, and internal ingest compatibility remain for Phase 06, as required by the phase boundary.
