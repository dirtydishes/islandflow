Islandflow implementation turn

Smart-flow alerts phase 02 moved runtime smart-flow to native projections.

Beads islandflow-ghce.2 Branch lavender/smart-flow-alerts-02-native-runtime 2026-06-26

Summary

Runtime smart-flow now has its own bus stream, persisted ClickHouse read model, compute emitter, API fetchers, and live-cache ingestion path. The old smart-money, classifier-hit, and alert emitters remain in place for comparison and rollback, but the API smart-flow routes no longer project from SmartMoneyEvent storage.

Implementation

Native stream

Added flow.smart_flow on SMART_FLOW as a derived JetStream stream.

Canonical storage

Added smart_flow_projections with full projection JSON plus ordering, refs, hypothesis, cluster, and abstention metadata.

Compute runtime

Added a deep runtime builder that turns a FlowPacket into native explainability projections through evidence, clusters, scoring, and projection in one call.

API and live

Smart-flow recent, history, replay, websocket, live-cache, durable rows, and option support now read native projections directly.
