Summary

Added named deterministic synthetic demo profiles and load profiles, wired them into hosted synthetic control, and moved the live options/equities synthetic emitters onto selected scenario fixtures instead of ambient runtime randomness.

Changes Made

  • Added the @islandflow/synthetic-market/profiles export with four DemoProfile definitions and three LoadProfile definitions.
  • Extended SyntheticControlState with demo_profile_id and load_profile_id, including defaults and metadata.
  • Updated options and equities synthetic adapters to replay deterministic named scenario fixtures selected by the active demo profile.
  • Exposed profile catalogs from the API synthetic admin status response and replaced raw preset/coverage controls in the internal web drawer with demo/load profile selectors.
  • Documented the new synthetic admin flags and profile behavior in README.md and .env.example.
  • Added focused package, replay, ingest, API, and web coverage around profile defaults, fixture determinism, live adapter playback, and status/control shape.

Context

Phase 05 called for deterministic named demo profiles and load profiles without changing scenario business semantics. The previous hosted control exposed lower-level knobs that could bias the synthetic scenario engine directly. This turn moves the public control surface up one level: demos choose known scenario runs, and load settings only alter cadence or repeated run count.

Important Implementation Details

  • market-command, event-response, quiet-range, and stress-tape map to named deterministic phase-03 scenario runs.
  • steady, active, and firehose adjust playback rate and repeated named-run volume only. They do not modify scenario labels, expectations, or classifier semantics.
  • Legacy SYNTHETIC_MARKET_MODE values still map to a default load profile before hosted control is changed, preserving local startup behavior.
  • KV decode now normalizes partial persisted control payloads so older hosted-control values get the new profile fields without breaking startup.
  • The web UI includes a local fallback catalog so the internal drawer remains stable if an older API response lacks profile metadata.
