Dashboard Command Page

Replaced the root overview surface with a dense Dashboard page based on the mock9 Market Command direction, using existing live terminal data and the current chart pane as the chart placeholder.

2026-06-13 03:38 ET Beads: islandflow-iil Branch: lavender/dashboard-command-page

Summary

The / route now presents as Dashboard instead of Home. It keeps the operational density of the mock9 command-board concept while using production data paths, existing terminal panes, and the current ChartPane for the chart slot.

Changes Made

  • Renamed the visible root route and nav item from Home to Dashboard.
  • Added dashboard-specific data derivation for command metrics, priority rows, decision levels, score states, and recent option contract context.
  • Replaced the old overview composition with a chart-first dashboard: command strip, chart plus decision levels, priority board, live context, event context, feed health, dark context, and replay mode.
  • Added responsive dashboard CSS so the dense desktop grid collapses into a single-column stack without page-level horizontal overflow.

Context

The mock9 design was only static study UI. This turn adapts the direction into the actual root terminal surface without adding a new route or changing the / URL. Decision levels are still derived from available packet and event context, because the backend does not yet emit formal invalidation-level objects.

Important Implementation Details

Data behavior. Dashboard priority rows merge alerts, smart-money events, and flow packets, then dedupe by symbol, packet, and read. Score states map into confirm, hold, watch, and reject.

Decision levels. Levels use packet underlying_mid when present, or derive an implied underlying level from contract strike and moneyness when possible.

Chart slot. The page uses the existing ChartPane and does not implement the final charting model yet.

Routing. / remains the canonical route. The component is now DashboardRoute, and legacy redirects to / still work.

Expected Impact for End-Users

Users land on a Dashboard page that behaves more like a command surface: chart first, active packet reads nearby, and contextual evidence grouped below. The page is more useful as a live operating view while still preserving access to focused options and news routes.

Issues, Limitations, and Mitigations

  • The chart pane can show Chart error: Failed to fetch when the API behind candles is unavailable. This is inherited from the existing chart pane and was left in place because charts are planned for later implementation.
  • Decision levels are derived context, not authoritative invalidation levels. The UI labels them as derived and falls back to an empty state when packet context is missing.
  • There are unrelated pre-existing dirty files in the worktree. They were not included in this implementation scope.
