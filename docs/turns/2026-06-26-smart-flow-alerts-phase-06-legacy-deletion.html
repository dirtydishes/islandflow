Smart-Flow Alerts Phase 06 Legacy Deletion

The final migration phase removes the old derived emitters, transport channels, API routes, websocket paths, storage helpers, ClickHouse table wiring, and product support fields. Runtime code now carries canonical smart-flow projections and smart-flow-alerts only; old derived history is intentionally dropped.

Beads islandflow-ghce.6 Branch lavender/smart-flow-alerts-06-legacy-deletion Spec docs/implementation/smart-flow-alerts/06-legacy-deletion-history-drop.md

Implementation

Legacy contracts and streams deleted

Removed old derived event schemas, bus subjects, stream catalog entries, live channel schemas, transition aliases, and tests that existed only for the retired event family.

Storage and API narrowed

Deleted legacy storage modules and helpers, removed API registration for old REST and websocket paths, and left canonical smart-flow plus smart-flow-alert history/replay fetchers intact.

Compute emits canonical outputs only

Removed classifier, old alert-scoring, parent-event compatibility, and smart-money evaluation paths from compute startup and packet flush handling.

UI support payloads finalized

Durable option rows no longer include classifier or smart-money support fields. Durable alert rows no longer expose legacy classifier-hit placeholders. Alert-bearing web surfaces populate from canonical alert rows and smart-flow projection support.

Reviewer repair

The review pass removed leftover legacy alert-row fields, deleted route aliases and raw durable fallback wiring, renamed active smart-flow support contracts, updated current anatomy copy, and added regressions that reject retired live channels.

Changed Files

  • Deleted legacy runtime modules and tests: storage helpers, API transition route helper, compute classifiers, alert scoring, parent events, smart-money evaluation, and their focused tests.
  • Updated canonical contracts in packages/types, bus stream definitions, API live state, durable rows, synthetic controls, ingest defaults, and focused tests.
  • Reviewer repair updated durable alert row contracts, API row composition, terminal route wiring, synthetic scenario manifests, smart-flow taxonomy names, live protocol tests, and current anatomy documentation.
  • Updated current-facing docs: README.md, docs/anatomy.html, docs/clickhouse-reset-runbook.md, and the smart-flow-alerts implementation guide.
  • Turn documentation: this closeout file in docs/turns.
