QA Durable Tapes Demo Route

Replaced the hidden /durable-tapes experiment with a public /qa route that renders every durable tape module as a deterministic QA demo matrix.

Summary

The new QA page is exposed from apps/web/app/qa/page.tsx and renders through QaRoute. The previous apps/web/app/durable-tapes/page.tsx entrypoint was deleted outright, and terminal route normalization no longer treats /durable-tapes as a known path.

/qa includes a section for each module. Every section has QA option chips and three pinned default templates: full width, 2/3 width, and 1/3 width. The chart section uses the reusable lightweight chart with fixture candles and the smart-flow direction bar layer in the lower pane.

Implementation Notes

  • Added apps/web/features/durable-tape/qa-page.tsx with deterministic fixtures for option prints, flow packets, equity prints, smart-flow projections, alerts, news, candles, and lower-pane smart-flow bars.
  • Moved terminal routing to /qa, added the QA nav item, and kept the route feature set on server-composed durable rows without chart stream subscriptions.
  • Removed the old durable tapes route file instead of leaving a redirect or compatibility page.
  • Reworked the page CSS from one dashboard-style composition grid into repeated QA sections with a stable 3-column template matrix that collapses to one column on mobile.
  • Added a default-on supportHydrationEnabled prop to OptionsTape so deterministic QA fixtures can opt out of backend support hydration without changing production behavior.
  • Updated route and browser transport tests so route behavior, live manifest scope, and localhost API URL handling all use /qa.
