Summary

This turn adds a pure smart-flow evidence extractor that converts existing FlowPacket inputs into a parsed FlowCandidate plus traceable evidence facts. It keeps the live smart-money compatibility path unchanged while making eligibility and quote quality explicit for the next smart-flow phase.

Changes Made

  • Extended smart-flow contracts with execution_context and eligibility_decision evidence fact kinds.
  • Strengthened FlowEligibility with accepted, rejected, and down_weighted status plus structured decision records.
  • Added services/compute/src/smart-flow-evidence.ts, a deterministic extractor for packet refs, member refs, quote refs, evidence quality, eligibility, and direct evidence facts.
  • Added infra-free tests for accepted quote-backed evidence, stale quote rejection, and wide/noisy down-weighting without hypothesis language.

Context

The selected issue was islandflow-zxh.2.1, the PR-sized child under smart-flow phase 02. The repository implementation README places smart-flow evidence clustering after synthetic manifests and before synthetic scenario labels. The phase document limits this child to eligibility and evidence facts, so clustering, feature-vector policy, hypothesis scoring, API, and UI work stayed out of scope.

Important Implementation Details

Traceability

Every candidate keeps packet IDs, member print IDs, and observation refs. Quote-backed facts include option NBBO and equity quote context refs when the packet contains enough quote data.

Eligibility

Thin or stale quote coverage can reject evidence. Partial staleness, wide spreads, noisy conditions, and inside-market context down-weight evidence without pretending to identify intent.

Boundary

The extractor is not wired to NATS, storage, alerts, or hypothesis scoring. It is a testable evidence layer for the next phase to cluster and summarize.

+        nbbo_spread: 0.04,
+        nbbo_coverage_ratio: 1,
+        nbbo_aggressive_ratio: 0.8,
+        nbbo_inside_ratio: 0.1,
+        underlying_bid: 449.9,
+        underlying_ask: 450.1,

Issues, Limitations, and Mitigations

  • The extractor uses current packet feature fields and does not add new raw quote trace IDs where upstream packets do not already expose them.
  • Thresholds are deterministic starter policy for eligibility, not historical calibration.
  • The output is not persisted or published yet, which keeps this child issue focused and avoids changing runtime behavior before clustering is scoped.
