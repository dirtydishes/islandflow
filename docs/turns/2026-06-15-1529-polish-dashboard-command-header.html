Summary

The dashboard route now has a single visible page title, Dashboard, a compact command rail below it, and a more readable virtualized Priority Board. The latest live pass removes the redundant packet column, strips the noisy flowpacket: prefix from secondary metadata, and changes vague decision language to review-oriented copy.

Changes Made

  • Replaced the old islandflow / Market Command header copy with a direct Dashboard page title.
  • Collapsed live status, last-seen time, scope, filters, and replay toggle into one compact rail.
  • Changed the vague default filter text from No active filter to Filters: All flow.
  • Clarified the standalone ticker value by labeling it as Scope, so SPY no longer appears without context.
  • Moved accepted live-preview styles into durable dashboard-specific CSS classes.
  • Promoted Dashboard to the route-level title and removed duplicated visible title copy from the command rail.
  • Added explicit narrow-screen rail overflow, mobile touch sizing, hover/focus feedback, and reduced-motion coverage.
  • Rebuilt the Priority Board as a four-column layout: event, evidence read, score, and review.
  • Moved packet IDs into secondary evidence metadata and removed the redundant flowpacket: prefix.
  • Added TanStack virtual scrolling to the Priority Board rows so the table can scroll smoothly without rendering every row at once.
  • Renamed visible dashboard Decision copy to clearer Focus, Review, and Review Levels labels.

Context

This started as an Impeccable live-mode polish pass on the root dashboard header. The follow-up pass moved the work onto a dedicated branch and tightened the whole / route hierarchy so the page title, command rail, and high-priority evidence table each do distinct jobs.

Important Implementation Details

  • The accepted live variant was carbonized by the helper, then cleaned into normal source by removing live markers, temporary wrappers, and scoped preview CSS.
  • The header keeps the existing Islandflow terminal identity: dark surfaces, mono metadata, sparse state color, flat borders, and compact density.
  • The command rail no longer inherits command-deck-header or compact-command-bar, which prevents generic mobile rules from changing its display mode.
  • The responsive rule hides the last-seen cell below the existing 1180px breakpoint, then uses horizontal overflow and 44px touch-height cells below 720px.
  • The Priority Board uses useVirtualizer with a fixed 56px estimate, overscan: 8, and stable row keys from the existing priority-row model.
  • The virtualized table keeps the header outside the scroll viewport so column labels remain stable while rows scroll.

Expected Impact for End-Users

The root dashboard should now scan with less duplication: the page title says where the user is, the rail shows operational state, and the Priority Board keeps evidence readable in a compact table. Users no longer have to decode a redundant packet column, and larger priority lists can scroll without turning the table into a cramped horizontal strip.

Issues, Limitations, and Mitigations

  • The command rail is intentionally dense. To protect narrower widths, the last-seen cell hides at the existing 1180px breakpoint.
  • The Priority Board row height is estimated at 56px for virtualization. If future row content becomes multi-line, the estimate may need to be revisited.
  • The in-app browser surface was unavailable in this session, so visual iteration happened through the live helper and system browser fallback.
  • The live helper stored a stale diagnostic about carbonize cleanup, but source inspection and build validation confirmed cleanup was completed.
  • For this follow-up pass, Playwright browser automation could not launch because the local Chromium and default Chrome binaries were missing. Final validation used route fetches, source inspection, detector output, focused tests, and production build validation.
