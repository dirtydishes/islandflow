Lightweight Charts Phase 03 Dashboard Replacement

Replaced the Market Command dashboard chart path with a terminal adapter feeding the reusable market-chart module. Terminal state, live and replay fetching, overlay range fetches, timeframe controls, and drawer callbacks stay in terminal-owned code.

Summary

OverviewRoute now renders TerminalMarketChartSection instead of ChartPane. The adapter normalizes candles, off-exchange overlay prints, smart-flow markers, legacy smart-money fallback markers, inferred dark markers, status, last-update metadata, and marker payload callbacks before handing props to MarketChart.

The reusable module gained generic overlay drawing for normalized overlay points. The market-chart package still has no terminal imports, and dashboard grid placement now uses .market-command-chart instead of an nth-child selector.

Implementation Notes

  • chart-adapter.tsx owns terminal fetches for candles and visible overlay ranges, live-session candle replacement, replay end_ts calculation, timeframe favorites, and marker payload routing.
  • buildTerminalMarketChartMarkers keeps smart-flow-first behavior by using the existing chart marker selector, then falls back to legacy smart-money only when no in-range smart-flow projection exists.
  • MarketChart renders normalized overlay points with a generic canvas layer using chart time and price coordinates.
  • Flat terminal section styling is preserved: radius 0, no decorative shadow, top and bottom rules, dense controls, and bg-pane-2 chart surface tone.
