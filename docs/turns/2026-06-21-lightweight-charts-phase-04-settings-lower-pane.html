Lightweight charts phase 04: settings and lower-pane modes

Added reusable chart settings state, versioned persistence, price renderer selection, lower-pane mode transforms, and a dashboard settings popover that stays inside viewport bounds on desktop and mobile.

Scope

Bead islandflow-mloi.4
Branch lavender/lightweight-charts-phase-04-settings-lower-pane
Phase 04 only

Changes Made

  • Added a reusable settings reducer, normalization, and local storage boundary that recovers from malformed storage and drops unknown extension keys unless the owning section is registered.
  • Added price renderer registry support for standard candles and Heikin Ashi, with the chart controller applying the selected transform immediately.
  • Added lower-pane mode registry support for flow direction, all flow, and volume, backed by pure tested transforms.
  • Added MarketChartSettings, a fixed-position settings popover with segmented mode controls, checkbox toggles, timeframe favorite rows, Escape close, outside-click close, and reset.
  • Integrated settings into the terminal chart adapter while keeping terminal data and callbacks outside features/market-chart.

Implementation Notes

  • The lower-pane smart direction mode prefers smart-flow projections and falls back to legacy smart-money events only when projections are absent.
  • Directional bars use signed values and explicit labels such as bullish flow direction or bearish flow direction, so red and green are not the only signal.
  • The settings UI uses restrained terminal styling: flat internal sections, compact controls, amber active/focus states, and no card framing around the chart module.
  • The dashboard can pass settings capabilities to hide or disable unsupported extension controls without changing the core settings menu structure.

+export const buildSmartDirectionBars = (projections, smartMoneyEvents, buckets) => {
+  const useSmartFlow = projections.length > 0;
+  const source = useSmartFlow ? projections : smartMoneyEvents;
+  return {
+    id: "smart-direction",
+    label: "Flow Direction",
+    kind: "signed-direction",
+    points: bucketRanges(toBuckets(buckets, [])).map(({ bucket, start, end }) => {
+      const signed = source
+        .filter((item) => item.source_ts >= start && item.source_ts < end)
+        .reduce((sum, item) => sum + magnitude(item) * directionSign(item), 0);
+      return { time: bucket.time, timestampMs: bucket.timestampMs, value: signed, label: "flow direction" };
+    })
+  };
+};
+
+export const MARKET_CHART_LOWER_PANE_MODE_REGISTRY = [
+  { id: "smart-direction", label: "Flow Direction", transformId: "buildSmartDirectionBars" },
+  { id: "all-flow", label: "All Flow", transformId: "buildAllFlowBars" },
+  { id: "volume", label: "Volume", transformId: "buildVolumeBars" }
+];
