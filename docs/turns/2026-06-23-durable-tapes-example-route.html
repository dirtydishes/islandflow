Durable Tapes Example Route

Added a hidden /durable-tapes QA route that composes the five durable tape domain modules from existing terminal state without adding a public navigation item or changing the domain module APIs.

ModulesOptions, flow packets, equities, news, alerts

Summary

The route lives in the existing terminal route module as DurableTapesExampleRoute and is exposed by a new Next entrypoint at apps/web/app/durable-tapes/page.tsx. It uses useTerminal() state and wires the existing terminal maps and focus callbacks into OptionsTape, FlowPacketsTape, EquitiesTape, NewsWire, and AlertsModule.

The route is intentionally hidden. NAV_ITEMS remains unchanged, while getRouteFeatures("/durable-tapes") subscribes to options, NBBO, equities, flow, news, alerts, and classifier hits. Chart and overlay feeds stay off.

Implementation Notes

  • Flow packets and equities use static durable tape sources built from state.filteredFlow and state.filteredEquities.
  • Options use state.filteredOptions, flow packet maps, NBBO maps, classifier decor, and shared flowFilters.
  • Alerts use state.filteredAlerts, state.selectedAlert, evidence maps, and alert callbacks for packet, contract, and equity focus.
  • News uses state.filteredNews, live session status, compact rails, and route-local history disabled so the page stays a composition proof over terminal live state.
  • The CSS uses a flat terminal grid with stable module heights, independent tape scroll regions, responsive template stepping, and no route-level horizontal scrolling.
