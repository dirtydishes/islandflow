Executive summary

No secret leak or dependency advisory was found, the raw API is private, and the desktop shell already uses strong renderer isolation. The dominant risk is instead a chain of trust-boundary mistakes: provider HTML reaches a regex sanitizer, the public edge permits plaintext and origin bypass, and several internal services assume every network peer is trusted.

What is unsafe today

Public identifiers reach ClickHouse through a broken string-escaping helper, and a harmless live probe confirmed that input can change the query shape. A compromised or malicious news payload can also execute script when you open a story. The production site answers over HTTP, the Next server listens on every interface, and public WebSockets accept unbounded subscription work. Electron 39 is end-of-support while loading this same remote content.

What changes before other users

Add a real identity/session boundary, make every privileged route authenticate the caller, enforce request/resource bounds, isolate service credentials, and run untrusted CI away from the privileged deployment runner. These are publication gates; they are not current cross-account failures because there are no other accounts.

F-00High / P1Public identifiers can inject ClickHouse SQL

IMPACT

A public caller can alter ClickHouse query structure through identifier parameters, bypass intended filters and potentially read or infer data accessible to the API’s broad default database user.

PRECONDITION

Access to an ordinary public lookup/history route. No account is required. Multi-statement DDL was not attempted; the confirmed issue is query-shape control inside a SELECT.

EVIDENCE

packages/storage/src/clickhouse.ts:686-693 — quoteString() doubles ' but does not escape \\
packages/storage/src/clickhouse.ts:868,1450,1869,2126 — helper feeds public identifier queries
services/api/src/option-queries.ts:62-86; services/api/src/index.ts:298-304,1720-1723 — public strings reach those queries without a safe character policy
Harmless live proof: generated SELECT added “7 AS marker”; ClickHouse returned the injected marker with HTTP 200

REMEDIATION

Stop interpolating string literals. Use ClickHouse typed query parameters such as {id:String} with param_id, add restrictive identifier schemas and quote/backslash regression tests, and replace the broad default database user with least-privilege read/write roles. See ClickHouse string syntax and query parameters. CWE-89 / OWASP A03.

F-01High / P1Stored XSS through provider-supplied news HTML

IMPACT

A crafted story body can execute script in the Islandflow origin when the news drawer opens, allowing same-origin API actions and access to browser-visible data.

PRECONDITION

Malicious or compromised provider content, an operator-configured compatible endpoint, or an internal publisher able to inject a news event.

EVIDENCE

services/ingest-news/src/index.ts:76-94 — provider content is persisted and published verbatim
packages/types/src/events.ts:243-258 — content_html is only z.string()
apps/web/features/terminal/components/ui-helpers.ts:137-159 — regex sanitizer
apps/web/features/terminal/components/drawers.tsx:80-86 — dangerouslySetInnerHTML sink
Reproduced: href=javascript:... and onclick = ... survived with sanitized:true

REMEDIATION

Replace the regex with a maintained parser-based allowlist; remove all attributes by default; parse and allow only required https:/http: links; sanitize before persistence and again at render; add regression payload tests and a restrictive CSP. CWE-79 / OWASP A03.

F-02High / P1Plaintext and direct-origin paths bypass the intended edge

IMPACT

Traffic can avoid an HTTPS-only/Cloudflare posture, exposing the app to network content injection, direct scanning, and edge-control bypass.

PRECONDITION

Ordinary network reachability. The domain’s HTTP endpoint returned 200, and the live Next listener is bound to all host interfaces.

EVIDENCE

F-18High / P1Public WebSockets allow unbounded subscription work

IMPACT

One anonymous connection can submit oversized subscription sets and identifiers that create snapshot/database work and persistent fanout keys. Handshake rate limits do not limit subsequent messages.

PRECONDITION

Public WebSocket access. Cross-origin browser sites can also connect because upgrades do not validate Origin; the data is public, so the primary risk is availability rather than confidentiality.

EVIDENCE

services/api/src/index.ts:1938-2023 — anonymous upgrades with no Origin validation
packages/types/src/live.ts:75-125,159-183 — subscription arrays/IDs lack practical maxima
services/api/src/index.ts:2068-2105; services/api/src/live.ts:947-1018 — each item can register work and trigger snapshots
Bun.serve sets no application-sized max WebSocket payload or per-socket work quota

REMEDIATION

Allowlist browser origins, set a small payload ceiling, cap subscriptions and identifier lengths per message/socket, enforce per-socket message/work quotas, close abusive connections, and apply backpressure. Origin checks are a browser-abuse control, not authentication. CWE-770.

F-04Medium / P2Rate limiting trusts unverified forwarded addresses

IMPACT

A caller that reaches the origin can rotate a spoofed first forwarded address, weakening quotas for REST, lookup, and WebSocket routes.

PRECONDITION

Direct-origin access or any proxy path that preserves a caller-supplied first X-Forwarded-For value.

EVIDENCE

services/api/src/rate-limit.ts:85-113 — first valid forwarded IP wins; socket peer is not checked
Live NPM proxy.conf — X-Forwarded-For $proxy_add_x_forwarded_for

REMEDIATION

Close direct origin access, overwrite forwarding headers at the first trusted edge, and resolve client identity only through an explicit trusted-proxy CIDR/hop policy. CWE-345/CWE-770.

Recommended remediation sequence

Fix the exploit chains before broad hardening. Each phase has an explicit gate so “safe enough for me” and “safe to invite someone” do not blur together.

PHASE 0
NOW
  • Replace interpolated ClickHouse strings with typed query parameters and add injection regression tests.
  • Replace the news HTML sanitizer and add hostile-content regression tests.
  • Bind web privately, close public 3000, and force HTTP→HTTPS.
  • Deploy CSP/report-only then enforce it; add baseline response headers.
  • Bound WebSocket payloads, subscriptions, identifiers, and per-socket work.
  • Upgrade Electron to a supported line before using/distributing the desktop shell.
GateNo confirmed browser-code-execution path; one HTTPS-only public entry; supported browser runtime.
PHASE 1
BEFORE USERS
  • Add owner/user identity and server-side authorization for every privileged action.
  • Fix trusted-proxy IP resolution and apply body/query/cardinality limits.
  • Split service identities/credentials; authenticate the internal data plane.
  • Backpressure provider ingestion and add timeouts, payload caps, and overflow telemetry.
GateA second user cannot invoke owner controls or exhaust shared resources through ordinary inputs.
PHASE 2
HARDEN
  • Sandbox systemd units and run containers non-root with read-only roots and limits.
  • Move untrusted PR CI to disposable isolation; pin actions, tools, and images immutably.
  • Lock/hash Python dependencies; upgrade supported datastore/container lines.
  • Build encrypted backups and prove restore; add SCA, image, secret, and SBOM gates.
GateOne component compromise does not automatically become full-host, all-credential, or unrecoverable-state compromise.

Positive controls

These controls are real and should be preserved. The audit did not treat absence of a finding as proof, but it did verify the following defensive properties.

  • bun audit --json returned no advisory matches for the locked JS graph.
  • bun pm untrusted found zero untrusted lifecycle-script dependencies.
  • No high-confidence secret, private key, or live credential fingerprint appeared in tracked files or reachable Git history.
  • Real env files are ignored from Git and Docker’s primary contexts; the current root env is mode 0600.
  • The raw API binds the Docker bridge and the public raw API host is disabled.
  • Native NATS, Redis, and ClickHouse listeners bind loopback only.
  • Production API rate limiting is enabled; synthetic admin is disabled.
  • CORS reflects an explicit allowlist and rejects unknown origins in focused tests.
  • Admin backend routes require a bearer token; the defect is in the optional web proxy caller boundary.
  • Desktop uses nodeIntegration:false, contextIsolation:true, sandbox:true, permission denial, and navigation guards.
  • JetStream retention is byte/time bounded and malformed bus payloads are schema-validated and terminally acknowledged.
  • Subprocesses use argv arrays rather than shell strings; provider keys are not passed in argv.
  • Docker and CI use bun install --frozen-lockfile; Docker workspace lock snapshots match.
  • Forgejo CI grants only contents: read and has timeout/concurrency cancellation.
  • No eval, new Function, command-injection path, or unsafe code-execution deserializer was found in assigned runtime surfaces.
  • Focused security controls passed 29/29 tests; public same-origin route probes passed.

Method and limits

This was a source-assisted, read-only security audit with safe live verification. It was not a destructive penetration test, did not fuzz production, and did not rotate secrets, restart services, alter firewall state, or exploit the confirmed XSS in a browser.
