Alerts Drawer Context Miss Repair

Fixed the alert detail hydrator path that rendered a raw {"error":"not found"} response as a flashing persisted-context drawer error when option-print evidence was unavailable.

Branchdashboard-v2
Beadsislandflow-g8wg
SurfaceAlerts detail drawer
Scope404 context misses, negative cache parity

What Failed

Flow-packet hydration already treated a 404 as missing evidence. The option-print batch lookup did not, so a missing persisted option-print response could throw the raw body into hydration.status.error.

The drawer rendered that state as Persisted context could not be loaded: {"error":"not found"}. In live dashboard state, repeated map or selection updates could retrigger hydration and make the message flash on and off above Primary evidence.

Implementation

  • Exported the alerts option-print fetch helper so the regression can exercise the real hydrator boundary.
  • Changed alert option-print hydration to convert 404 into missing refs instead of a fatal status error.
  • Changed the terminal hydration scheduler to treat the same 404 response as an empty lookup, preserving its negative-cache behavior.
  • Left non-404 failures as real errors so service outages and invalid responses still surface through the existing error/backoff paths.
