Repository turn note

QA ingest investigation and Eastern time display

Investigated the live QA deployment around the reported 10:30am stop, found ingestion was still running, and changed the web UI to present market timestamps in America/New_York time so UTC chart labels do not read like local stops.

Branchmain
Beadsislandflow-s1a7
ScopeQA triage, chart ticks, tape timestamps

Incident Readout

No ingest stop at 10:30 EDT

Journals for ingest and compute continued through 2026-06-26 10:30 EDT. ClickHouse 5-minute buckets also had normal options, equities, and candle rows in the matching 2026-06-26 14:30 UTC bucket.

The 2:30 label was UTC

The chart-visible 14:30 or 2:30 value maps to 10:30 EDT on June 26. The data path stores timestamps in UTC milliseconds; the UI was formatting several surfaces with viewer-local defaults or chart defaults.

Synthetic mode is expected

Both options and equities ingest services are configured with synthetic adapters on this QA instance. Fresh public options, equities, candles, and flow packet endpoints returned current rows during inspection.

Separate noisy service issue

The wrapper islandflow-clickhouse.service is restart-looping because distro clickhouse-server.service already owns the port. The active ClickHouse server is still serving the app successfully.

Changes Made

  • Added a shared apps/web/features/time-format.ts module that always formats dates and times in America/New_York.
  • Wired market chart tick labels and chart hover labels through the Eastern formatter. Hover text includes the actual daylight/standard abbreviation from Intl, for example EDT in June.
  • Moved tape rows, alerts, flow packet metadata, news timestamps, terminal status text, and replay controls onto the same Eastern formatter.
  • Added tests that pin the reported 2026-06-26T14:30Z timestamp to 10:30 AM EDT and guard Eastern-day comparisons around UTC midnight.

+export const EASTERN_TIME_ZONE = "America/New_York";
+export const EASTERN_TIME_LABEL = "ET";
+
+const formatterCache = new Map<string, Intl.DateTimeFormat>();
+
+const getFormatter = (
+  options: Intl.DateTimeFormatOptions,
+  locale = "en-US"
+): Intl.DateTimeFormat => {
+  const normalizedOptions = { ...options, timeZone: EASTERN_TIME_ZONE };
+  const key = JSON.stringify(
+    Object.entries(normalizedOptions).sort(([left], [right]) => left.localeCompare(right))
+  );
+  const cached = formatterCache.get(key);
+  if (cached) return cached;
+  const formatter = new Intl.DateTimeFormat(locale, normalizedOptions);
+  formatterCache.set(key, formatter);
+  return formatter;
+};
+
+export const formatEasternTime = (
+  ts: number,
+  options: Intl.DateTimeFormatOptions = {},
+  locale = "en-US"
+): string => getFormatter({ hour: "numeric", minute: "2-digit", ...options }, locale).format(new Date(ts));
+

+export const formatChartTickTime = (value: ChartTimeLike, tickMarkType?: unknown): string => {
+  const timestampMs = chartTimeToMs(value);
+  if (timestampMs === null) return "";
+
+  switch (normalizeTickKind(tickMarkType)) {
+    case "year":
+      return formatEasternDate(timestampMs, { year: "numeric", month: undefined, day: undefined });
+    case "month":
+      return formatEasternDate(timestampMs, { year: undefined, month: "short", day: undefined });
+    case "day":
+      return formatEasternDate(timestampMs, { year: undefined, month: "short", day: "numeric" });
+    case "time-with-seconds":
+      return formatEasternTime(timestampMs, { hour: "numeric", minute: "2-digit", second: "2-digit" });
+    case "time":
+      return formatEasternTime(timestampMs, { hour: "numeric", minute: "2-digit" });
+  }
+};
