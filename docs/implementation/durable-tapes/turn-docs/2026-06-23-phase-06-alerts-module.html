Implementation Notes

  • Added apps/web/features/alerts/ with AlertsModule, alert columns/templates, array/history source helpers, evidence hydration, detail rendering, and tests.
  • Moved alert context helpers behind the alerts module while keeping terminal exports as compatibility re-exports for existing tests and callers.
  • Replaced the selected-alert terminal drawer path with AlertDetailDrawer, so selected alert evidence loading is module-owned instead of terminal global drawer state.
  • Added typed terminal adapters for alert packet focus, contract focus, and equity focus. Packet focus seeds the options contract scope from member trace IDs; contract focus uses the existing options focus callback; equity focus sets the equity instrument context.
  • Embedded AlertsModule in the dashboard event-context pane using the phase one-third template. The existing smart-flow, dark, and news context rows remain below the alert tape.

Reviewer Caveats

  • Phase 06 does not add a standalone /alerts route; route composition is left for Phase 07.
  • Alert detail can focus equity context from the inferred alert underlying, because alert evidence currently resolves packet and option-print refs rather than equity print refs.
  • The terminal still prefetches visible dashboard alert evidence for hot-row support; the selected alert drawer no longer depends on terminal-owned context status or selected evidence state.
