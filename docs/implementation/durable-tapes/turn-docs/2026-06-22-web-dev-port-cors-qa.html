Web Dev Port CORS QA

Made web-only hosted-backend QA usable when port 3000 is occupied by giving bun run dev:web a supported alternate port path and locking the API CORS expectation in tests.

Root Cause

Hosted API CORS reflects http://localhost:3000 and http://localhost:3100, but not http://localhost:3001. The web dev runner always started Next on 3000, so a worker with another worktree on that port had no documented first-class fallback and manually landed on the unsupported 3001 origin.

Fix

  • Added apps/web/scripts/dev-config.ts to resolve the web dev API URL and port from environment values.
  • WEB_DEV_PORT is the explicit knob; PORT also works for environments that already set it.
  • The supported hosted-backend fallback is NEXT_PUBLIC_API_URL=https://api.flow.deltaisland.io WEB_DEV_PORT=3100 bun run dev:web.
  • The runner warns when the hosted API is paired with unsupported local ports such as 3001.
  • .env.example now documents the full hosted QA fallback command with NEXT_PUBLIC_API_URL=https://api.flow.deltaisland.io and WEB_DEV_PORT=3100, and API tests lock the default 3100 allowlist expectation.

+export const resolveWebDevConfig = (env) => {
+  const configuredPort = firstConfiguredValue([
+    ["WEB_DEV_PORT", env.WEB_DEV_PORT],
+    ["PORT", env.PORT]
+  ]);
+  const port = configuredPort
+    ? parseWebDevPort(configuredPort.value, configuredPort.source)
+    : DEFAULT_WEB_DEV_PORT;
+  const { apiUrl, apiUrlSource } = resolveApiUrl(env);
+  return {
+    apiUrl,
+    apiUrlSource,
+    hostedApiCorsWarning: resolveHostedApiCorsWarning(apiUrl, port),
+    port,
+    portSource: configuredPort?.source ?? "default"
+  };
+};
