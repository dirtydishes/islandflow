Durable Tape Empty History Backfill

Added a shared initial-history cursor path so durable tapes can request ClickHouse history after a filter or scope change leaves the live head empty.

Beadsislandflow-m2m7
Scopeshared durable-tape blocker
Timestamp2026-06-23 EDT

Summary

DurableTapeSource now has an optional getInitialHistoryCursor(query) hook. DurableTape uses it only when the composed row stack is empty, so existing non-empty history paging still derives cursors from visible rows and the virtual tail gate.

Options, flow packets, equities, and news wire default sources opt into the shared path with a current-time history cursor. The adapters still use their existing loadOlder implementations and filter semantics; no domain module cloned history mechanics.

Behavior

  • Empty visible/live result sets can seed one initial history request after query or source reset.
  • Once rows exist, row-derived cursor paging, scroll hold, and jump-to-live behavior are unchanged.
  • Repeated nextCursor values are treated as exhausted at the shared layer to avoid loader loops.
  • Sources that do not provide getInitialHistoryCursor keep the previous default behavior.

Reviewer Pass

Reviewer repair commit 3f9d2c8 found and fixed a shared state-machine race: a history request started for one query or source could resolve after a reset and append stale rows into the new tape state. DurableTape now versions history loads by query/source generation and discards stale completions before mutating history items, cursors, exhausted state, or loading state.

No domain-specific workaround was added. The repair stays in the shared durable-tape loader, so options, flow packets, equities, news wire, and the upcoming alerts module use the same reset-safe history mechanic.
