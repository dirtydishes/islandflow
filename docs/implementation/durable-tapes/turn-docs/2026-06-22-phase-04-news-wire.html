What Changed

  • Added a reusable NewsWire module with source and symbol filters, mapped/unmapped and updated-only controls, hover metadata, module-owned story detail, responsive templates, and live-first history loading.
  • Replaced the dedicated /news composition with NewsWire while keeping terminal route state as a thin adapter for live stories, status, active ticker scope, and live-enabled state.
  • Moved default visible columns to the Phase 04 matrix: full TIME | SOURCE | SYMBOLS | HEADLINE, two-thirds TIME | SYMBOLS | HEADLINE, and one-third TIME | HEADLINE. Summary, provider metadata, updated state, raw mapped symbols, content HTML, and source link now live in hover/detail by default.
  • Kept the existing terminal news helpers compatible while narrowing the public news-wire barrel to the component boundary.
  • Reviewer repairs added explicit durable-tape row activation, hardened story body rendering, fixed filtered history paging, constrained the full-route tape height, and filed follow-up islandflow-azga for backend news cursor ordering.

Reviewer Findings And Repairs

  • P1 fixed: filtered history could stall when a backend page had no client-side matches. fetchNewsWireHistoryPage now scans successive /history/news pages until it finds matching stories or reaches server exhaustion, with a repeated-cursor guard and focused tests.
  • P1 fixed: row focus opened detail by side effect. DurableTape now has a separate onActivate callback, and NewsWire opens detail only on click, Enter, or Space. Focus still supports hover/inspection.
  • P1 fixed: the full news route let the virtual list expand the document, which could render thousands of rows and continuously trigger history loading. The route now constrains the module height and the durable-tape history gate ignores all-rendered measurement states.
  • P2 fixed: provider story bodies were rendered through a regex sanitizer and dangerouslySetInnerHTML. The new module now renders decoded plain text from content_html, deleting the unsafe HTML injection path.
  • P2 fixed: the reusable module leaked terminal replay vocabulary through mode. The public prop is now liveEnabled, and the route owns mapping terminal mode to the module.
  • P2 fixed: filter focus styling and mobile detail wrapping were hardened with a distinct focus box shadow and overflow-wrap: anywhere.
  • P2 follow-up filed: backend recent-news ordering and /history/news cursor ordering differ for same-timestamp stories. This is tracked as islandflow-azga because the repair belongs in storage/API tests, not this UI module PR.

+export const NEWS_WIRE_HISTORY_ENDPOINT = "/history/news";
+export const NEWS_WIRE_HISTORY_BATCH = 200;
+
+export const buildNewsWireHistoryUrl = ({
+  cursor,
+  limit = NEWS_WIRE_HISTORY_BATCH,
+  buildApiUrl = buildNewsWireApiUrl
+}: {
+  cursor: DurableTapeCursor;
+  limit?: number;
+  buildApiUrl?: NewsWireApiUrlBuilder;
+}): string => {
+  const url = new URL(buildApiUrl(NEWS_WIRE_HISTORY_ENDPOINT));
+  url.search = new URLSearchParams({
+    before_ts: String(cursor.ts),
+    before_seq: String(cursor.seq),
+    limit: String(limit)
+  }).toString();
+  return url.toString();
+};
