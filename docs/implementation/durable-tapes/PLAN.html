Implementation Plan

Durable reusable tape modules

Epic islandflow-h9c0 Phase 00 docs Updated 2026-06-22

Islandflow needs one shared durable tape foundation and a set of domain modules for options, flow packets, news, equities, and alerts. The shared foundation owns live hot heads, ClickHouse history, virtual scrolling, scroll hold, feature flags, templates, and hover primitives. Domain modules own market meaning and route composition.

The UI direction is restrained, dense, flat, and table-first. Production templates must not require horizontal scrolling. Omitted context belongs in hover, focus, settings, or module-owned detail surfaces.

Product Rules

Use $impeccable for UI and UX
No horizontal scroll

Templates step down by container width. The registry may be rich, but visible columns stay small.

Scroll hold is default

Scrolling away from the live head pauses visible insertion, not ingestion. Jump-to-live flushes queued rows.

Signal as row treatment

Signals and hypotheses tint rows and appear in hover/detail. They do not consume a default column.

Small settings gear

Filters live in a popout rendered outside virtual scroll containers so it cannot be clipped.

Domain modules stay separate

Options does not own alerts, alerts do not own packet UI, and routes compose modules.

Historical rows are first-class

ClickHouse history renders like valid tape data. Feed health and row validity are separate concepts.

Target Module Stack

Shared mechanics, domain meaning
features/durable-tape

Feature flags, templates, virtual table shell, hot/history merge, scroll hold, settings and hover primitives.

features/options-tape

Signal-first options tape, contract focus, packet-print focus, settings filters, readable contracts.

features/flow-packets

Packet summaries, packet detail, packet focus callbacks, and durable packet history.

features/news-wire

Live-first news, source and symbol filters, headline-first templates, story detail.

features/equities-tape

Equity prints, ticker scope, venue and off-exchange context, chart-independent tape behavior.

features/alerts

Alert list, alert detail, evidence hydration, and links to options, flow, and equities modules.

Orchestrator Loop

Selector subagent, worker thread, callback
Find work from Beads

The orchestrator reads IMPLEMENT.md, then asks a narrow subagent to run bd ready, inspect the durable-tapes issue, and report the next phase.

Create a worker thread

The orchestrator starts a worker thread with the full phase doc, Beads issue, gates, branch posture, and the orchestrator thread ID.

Require one callback

The worker messages the orchestrator thread only when complete or genuinely blocked, with changed files, tests, Beads updates, push or PR state, and follow-up issues.

Review separately

Implementation phases get a separate ambitious reviewer thread after the worker completes or opens the assigned PR.

Reviewer Thread Standard

Push for deletion
Review ambitiously

Reviewer threads should not stop at minor cleanup. They should look for the simpler shape that makes the change feel inevitable.

Delete complexity

Prefer findings that remove helpers, branches, conditionals, modes, layers, or legacy coupling instead of rearranging them.

Use the existing architecture

Push code toward route composition, domain modules, and one shared durable-tape mechanic instead of per-domain clones.

Name the better shape

Lead with behavioral or architectural risk, then state the code-judo move that would make the implementation smaller.
