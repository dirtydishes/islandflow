API Private Edge Phase 00 Baseline

Baseline for islandflow-hnbk.1. This phase records the current public edge posture before private-edge changes: the hosted app remains public, the raw API host is still reachable, production browser artifacts still expose <raw-api-origin>, and the same-origin app proxy matcher misses active browser routes. No runtime, deployment, env, DNS, firewall, NPM, or product code behavior changed.

Risks And Constraints

  • Raw API host and direct port 4000 currently return API JSON. Closing that exposure is Phase 04, after same-origin coverage and rate limiting are in place.
  • The app-origin proxy matcher currently misses active browser routes. A production build with blank NEXT_PUBLIC_API_URL would still fail those flows until Phase 02 updates NPM and probes.
  • Websocket upgrades bypass the current CORS response path. Origin policy and rate limiting should be assessed in Phase 03, with priority on /ws/live, lookup, evidence, history, replay, candles, and range routes.
  • NPM admin is bound on the host. This baseline did not change or test admin-plane access control; treat that as protected infrastructure during deployment phases.
  • Historical docs and baselines contain concrete production domains. Phase 01 should focus active surfaces first so the private-edge stream stays reviewable and does not rewrite unrelated history.
  • Live ClickHouse was observed in auto-restart during the baseline. That does not block the docs-only artifact, but later production probes should account for backend health before interpreting market-data route results.
