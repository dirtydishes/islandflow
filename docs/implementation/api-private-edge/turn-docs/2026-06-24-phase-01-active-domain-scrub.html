API Private Edge Phase 01 Active Domain Scrub

Phase 01 for islandflow-hnbk.2 removes active defaults and examples that taught local developers, agents, probes, or deployment helpers to reach concrete production API domains. Local development now defaults to http://127.0.0.1:4000, production app/API origins are explicit operator inputs or placeholders, and the desktop app-origin allowlist remains documented as a product trust-policy constant.

Scope And Inputs

phase

Read IMPLEMENT.md and 01-active-domain-scrub.md. Phase posture is documentation/config hygiene, with no Phase 02 transport behavior, rate limiting, raw API closure, or deployment rollout.

issue

bd show islandflow-hnbk.2 confirmed the ready Phase 01 task. bd update islandflow-hnbk.2 --claim claimed it before edits.

helpers

Five read-only helpers inventoried docs/prompts, env/deploy surfaces, web/dev probes, tests/constants, and repo-wide hostname defaults. The worker resolved the scope and made all edits.

style

This document follows the Islandflow product register: compact, evidence-first, restrained, and operational. Diff rendering uses a standard HTML fallback plus browser-side PatchDiff.

Risks And Follow-Up

  • Production operators must now set real public app/API verification inputs explicitly for deploy, rollback, and production smoke probes. This is the intended Phase 01 posture, but reviewer should confirm the resulting messages are clear enough for emergency operations.
  • The desktop production URL remains concrete as a trust-policy product constant. It is documented in source as not being an API default.
  • Historical turn docs and baseline JSON/HTML artifacts still contain old concrete domains. They were not rewritten because Phase 01 explicitly excludes historical artifacts unless linked as current instructions.
