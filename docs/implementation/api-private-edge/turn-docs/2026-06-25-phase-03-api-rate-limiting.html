API Private Edge Phase 03 API Rate Limiting

Phase 03 for islandflow-hnbk.4 adds env-gated API throttling before route handling, with separate REST, evidence lookup, and websocket upgrade buckets for the hosted app API surface.

Outcome

Rate limit gate

Added a fixed-window in-memory limiter that is disabled by default, enabled through env, and applied before REST dispatch, body parsing, and websocket upgrades.

Route buckets

/health and preflight are exempt. Ordinary reads, evidence lookup routes, synthetic admin routes, and /ws/* upgrades are categorized with low-cardinality labels.

Client keying

Valid X-Forwarded-For values are preferred, then X-Real-IP, then Bun socket peer, with malformed headers falling back safely instead of breaking requests.

Observable rejection

Rejections return bounded JSON 429 responses with Retry-After and emit coarse bucket/category metrics and logs without query strings, tokens, trace IDs, or raw client IPs.

Risks

  • The limiter is in-memory and per API process. That matches the current single-process phase scope, but it is not a distributed quota.
  • Forwarded client headers remain spoofable while the raw API host is still reachable. Phase 04 owns raw host closure.
  • The metrics facade remains a no-op unless the runtime wires a metrics emitter. The code uses the existing API metrics surface.
