API Private Edge Phase 04 Raw API Host Closure

Phase 04 for islandflow-hnbk.5 closes the direct raw API proxy host by default while preserving app-origin REST and websocket routing through <production-app-origin>.

Outcome

Raw host closed

The NPM raw API proxy host is disabled in database state, websocket upgrades are disabled there, and the generated raw proxy config is removed so stale config cannot keep serving direct API traffic.

Same-origin preserved

The app proxy host keeps the full API prefix matcher for REST and /ws/live, and generated app config is synchronized with the durable DB matcher when the helper runs.

Rollback explicit

Raw public API reopen now requires ISLANDFLOW_RAW_API_MODE=temporary-open or --raw-api=temporary-open. Runtime rollback back to Docker does not reopen it by default.

Live verified

Redacted live probes showed internal API health still works, app-origin API routes and websocket pass, and raw-host health, market REST, and websocket upgrade probes do not reach the API.

Risks

  • Raw-host closure now depends on NPM DB state plus generated config sync. The helper covers both, and live NPM was updated through the container because the host-mounted DB is root-owned.
  • Raw host behavior is currently TLS/SNI failure, not a pretty 404 page. This satisfies the closed/no-JSON/no-upgrade posture.
  • Do not set DEPLOY_PUBLIC_API_HEALTH_URL in normal same-origin deploys. It is now only appropriate for a deliberate temporary raw-host reopen.
