API Private Edge Phase 05 Production Rollout Closeout

Phase 05 for islandflow-hnbk.6 deployed and verified the final private-edge posture: browsers use <production-app-origin>, the raw API host is closed, and direct public :4000 no longer reaches the API.

Outcome

Production web rebuilt

The live web build was rebuilt with NEXT_PUBLIC_API_URL=, then web and API user units were restarted. The hosted UI returns a successful response from <production-app-origin>.

Internal API selected

Production now uses an internal NPM bridge origin for server-only API access. The API binds to that internal host instead of wildcard public interfaces.

Rate limits enabled

Phase 03 fixed-window limits are enabled in the live env with the conservative production values from the implementation plan.

Closeout documented

Native unit defaults and deployment docs now steer future installs away from wildcard API binds and raw public API examples.

Risks

  • The live native API service relies on a server-local env value for API_HOST that points at the NPM bridge host. Keep that value aligned with the active NPM network if Docker networks are rebuilt.
  • Raw host behavior is TLS/SNI failure rather than a friendly 404 page. That is acceptable for the closed posture because it returns no API JSON and no websocket upgrade.
  • Shutdown logs still show a Redis client closed error during API restarts. It did not prevent startup or live verification, but it remains unrelated operational noise.
