Resolved posture

What this stream will make true

The public app stays usable. The raw API host stops being a product surface. Direct operational access moves to SSH, VPN, or server-local checks.

Public surface

The browser uses app-origin REST and websocket paths in production. The public bundle should not need or advertise the raw API origin.

Private infrastructure

The backend API remains reachable from the app edge and from trusted operational paths, not from public direct API clients.

Abuse posture

Same-origin endpoints remain public enough for the hosted UI. Phase 03 adds rate limits and telemetry as the v1 control layer.

Documentation rule

Active docs, defaults, scripts, and probes use placeholders such as <production-app-origin> and <raw-api-origin>.

Selector, worker, reviewer, closeout

The orchestrator owns sequence and final state. Helper threads receive bounded context and call back once with evidence.

Implementation swarm topology

Selector, scouts, worker, reviewers

Broad phases can use many bounded helpers around one accountable implementation owner. Helpers increase coverage; they do not split ownership.

Stage Delegation shape Owner and exit
Selector agent Picks the next ready islandflow-hnbk.* issue from Beads after reading the implementation index and phase doc. Reports the next ready task, dependency state, safe parallelism, and blocking contracts.
Scout swarm, 6-10 read-only agents Inspects different slices in parallel: repo hostname/default inventory, web transport URL builders, API route and websocket inventory, NPM/deployment helper behavior, live server/env baseline, test coverage gaps, security/threat-model review, and docs/probe references. Returns findings only. Scouts do not edit files, update Beads, create branches, commit, push, open PRs, or contact the orchestrator independently.
Single implementation worker Owns the branch, edits, tests, commit, PR, Beads state, phase turn doc, and callback. Uses scout outputs as inputs, resolves conflicts, and calls back once when complete, PR-ready, or genuinely blocked.
Review swarm, 3-6 agents Runs bounded review after the PR or worker callback: security reviewer, deployment reviewer, frontend transport reviewer, API/rate-limit reviewer, docs/Beads reviewer, and CI/log reviewer. Produces findings and safe repair recommendations without widening the phase.
One lead reviewer Consolidates review-swarm findings, performs safe repairs, waits for CI/local gates, and updates the existing phase turn doc. Calls back once when review, repairs, gates, Beads state, and push state are resolved.

1. One owner

The implementation worker owns the branch, edits, Beads state, commit, PR state, phase turn doc, and final callback.

2. Bounded helpers

The worker may launch helper subagents when useful. Broad inventory phases can use a scout swarm; narrow phases can use none.

3. Allowed helper work

Helpers may inspect, inventory, run focused checks, review risk, map test gaps, and propose patches or follow-ups.

4. Hard limits

Helpers must not mutate tracked files, update Beads, create branches, commit, push, open PRs, contact the orchestrator, or make final scope decisions.

Interface changes

Environment and routing contracts

These contracts are intentionally small. They give workers enough direction without forcing a larger auth product into this stream.

Public browser config

  • NEXT_PUBLIC_API_URL remains an explicit override.
  • Production builds should work with it blank.
  • Nonlocal browser fallback resolves to app-origin paths.

Server-only API config

  • ISLANDFLOW_INTERNAL_API_URL points to the backend from the web server.
  • Synthetic admin proxy uses the internal URL plus its existing token.
  • Direct public API clients are not a v1 interface.

App-origin route matcher

^/(ws|replay|prints|joins|nbbo|quotes|dark|flow|candles|history|news|lookup|option-prints|equity-joins)(/|$)

Acceptance evidence

What proves the stream is done

Closeout needs both repo evidence and live evidence. A green code path is not enough if the public edge still exposes the raw API host.

Final evidence checklist

  • Active docs, defaults, and probes do not use concrete production API domains as defaults.
  • Production web bundle does not contain the raw API origin.
  • Hosted UI loads and live data works through app-origin REST and websocket paths.
  • Raw public API host returns closed or 404-style behavior for market-data routes.
  • Direct public port 4000 remains unreachable.
  • Server-local API health works over SSH or the chosen trusted path.
  • bd dolt push, git push, and branch status verification succeed.
