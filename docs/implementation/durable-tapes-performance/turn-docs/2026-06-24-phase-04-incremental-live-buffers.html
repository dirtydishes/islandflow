Phase 04 · June 24, 2026

Incremental Live Buffers

Replaced the durable-tapes live-session hot update path with keyed bounded buffers and frame-batched websocket event application. Snapshot resets still preserve the existing cursor and reset behavior, while event batches now upsert into per-channel buffers before updating React state.

Beads issue islandflow-ze79.5

Probe output baselines/phase-04-local-hosted-api.json
Follow-up islandflow-ze79.9

Implementation

  • Added createLiveWindowBuffer, a keyed bounded window that hides duplicate handling, ordered insertion, snapshot rebuilds, and overflow eviction.
  • Added parity tests covering insertion, duplicate update, out-of-order input, reset, eviction, trim callbacks, and equal-cursor batch stability.
  • Replaced the durable-tapes live-session event path's mergeNewestWithOverflow calls with per-channel buffers for options, NBBO, equities, flow, alerts, news, inferred dark, chart, and evidence channels.
  • Batched websocket event messages by subscription until the next animation frame, then applied each batch through the same buffers and committed last-event state once per frame instead of once per message.
