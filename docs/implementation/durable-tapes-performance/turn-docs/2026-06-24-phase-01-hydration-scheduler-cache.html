Phase 01 / Durable Tapes Performance

Shared Hydration Scheduler And Caches

This pass centralizes terminal support/evidence hydration so live updates stop aborting and restarting the same lookups while preserving the existing durable pane content.

Beads issue islandflow-ze79.2
Branch lavender/ze79-phase-01-hydration-scheduler
Scope frontend request behavior only

Changed Files

  • apps/web/features/terminal/hydration-scheduler.ts: new shared scheduler for option support, option print, and flow-packet hydration.
  • apps/web/features/terminal/state.tsx: options support, selected smart-flow, selected smart-money, selected classifier, and visible alert evidence effects now depend on stable missing-id keys and call the scheduler.
  • apps/web/features/terminal/hydration-scheduler.test.ts: coverage for sorted keys, batching, in-flight reuse, negative cache expiry, and endpoint backoff.
  • docs/implementation/durable-tapes-performance/baselines/phase-01-before-local-hosted-api.json and docs/implementation/durable-tapes-performance/baselines/phase-01-after-local-hosted-api.json: before/after Phase 00 probe captures.

Implementation Notes

  • Added a module-level HydrationScheduler with TTL-bound positive caches, short negative caches, max-size eviction, per-endpoint backoff, and in-flight reuse.
  • Removed component-owned abort controllers from shared-worthy support/evidence lookups. React cleanup now only prevents stale component state writes; it does not cancel shared fetches other callers can reuse.
  • Batched /lookup/options-support on a 3-second support window and kept print/packet evidence hydration on the short scheduler window for selected drawers.
  • Replaced effect dependencies on live arrays, resolved maps, and fresh Sets with stable sorted missing-id summary keys for the network effects.

Known Risks

  • The after probe still reports TaskDuration 61.54s and ScriptDuration 57.37s against 45s / 35s budgets. The support/evidence storm is materially bounded, but broader live recomputation remains for the downstream pane-scoped state and live-buffer phases, especially islandflow-ze79.4.

+const DEFAULT_SUPPORT_BATCH_DELAY_MS = 3_000;
+const DEFAULT_NEGATIVE_TTL_MS = 30_000;
+const DEFAULT_MAX_ENTRIES = 2_000;
+export const stableHydrationKey = (ids) => normalizeHydrationIds(ids).join("\n");
+export class HydrationScheduler {
+  async requestOptionSupport(input: OptionSupportRequest): Promise<OptionSupportResult> {
+    const traceIds = normalizeHydrationIds(input.traceIds ?? []);
+    const nbboContext = normalizeNbboContext(input.nbboContext ?? []);
+    await Promise.all([
+      ...traceIds.map((traceId) => this.queueSupportTrace(traceId)),
+      ...nbboContext.map((context) => this.queueSupportNbbo(context))
+    ]);
+    return this.collectOptionSupportResult(traceIds, nbboContext);
+  }
+  async requestOptionPrints(traceIds: string[]): Promise<OptionPrintLookupResult> {
+    const ids = normalizeHydrationIds(traceIds);
+    await Promise.all(ids.map((traceId) => this.queueOptionPrint(traceId)));
+    return { prints, missingTraceIds };
+  }
+}
+export const terminalHydrationScheduler = new HydrationScheduler();
