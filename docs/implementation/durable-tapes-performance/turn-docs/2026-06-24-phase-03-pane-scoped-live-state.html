Durable Tapes Performance Phase 03: Pane-Scoped Live State

Split the durable-tapes route away from one broad terminal state consumer and routed each pane through a stable terminal store selector. The route keeps one live websocket, keeps the shared hydration scheduler, and preserves the existing pane UI.

Timestamp2026-06-24

Summary

  • Added a stable terminal state store and useTerminalSelector so route children can subscribe to selected slices instead of the whole terminal context value.
  • Moved /durable-tapes into five memoized pane adapters: options, flow, equities, alerts, and news.
  • Kept flow and equities on static array-backed sources fed by the existing shared /ws/live session, avoiding duplicate websocket connections.
  • Made live feed timestamps channel-scoped and memoized static composed live feeds so unrelated channel updates do not manufacture new pane props.
  • Kept alert evidence maps empty while no alert detail is selected, so options print map churn does not wake the alert pane in its normal closed-detail state.
  • Moved closed terminal drawers behind their own selector so the shell no longer renders drawer JSX against the full state on every live tick.
