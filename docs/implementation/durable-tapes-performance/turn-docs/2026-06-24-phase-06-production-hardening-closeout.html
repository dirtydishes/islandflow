Phase 06 / durable-tapes performance

Production hardening and closeout guardrails

This turn adds repeatable production smoke checks, richer browser-probe observability, API latency timing, and a hardened durable-tapes fallback flag. Source guardrails are in place; deployed closeout remains blocked by native API/web state.

What Changed

  • Added scripts/probes/durable-tapes-production-smoke.ts for deployed web route, public API routing, API health, durable-row websocket, support lookup, and by-trace miss latency checks.
  • Extended the CDP browser probe with support/evidence origin and content-type distributions, HTML/non-JSON counters, latency summaries, and options/alerts pane row budgets.
  • Added api.lookup.options_support_ms timing around /lookup/options-support.
  • Hardened /durable-tapes so raw options/alerts fallback channels are off by default. Set NEXT_PUBLIC_DURABLE_TAPES_RAW_FALLBACK=1 to opt back in during an incident.
