What Changed

  • Added scripts/probes/durable-tapes-production-smoke.ts for deployed web route, public API routing, API health, durable-row websocket, support lookup, and by-trace miss latency checks. The routing fallback now requires both same-origin REST proxy paths to return bounded JSON if the public bundle omits the API origin.
  • Extended the CDP browser probe with support/evidence origin and content-type distributions, HTML/non-JSON counters, latency summaries, and options/alerts pane row budgets.
  • Added api.lookup.options_support_ms timing around /lookup/options-support.
  • Hardened /durable-tapes so raw options/alerts fallback channels are off by default. Set NEXT_PUBLIC_DURABLE_TAPES_RAW_FALLBACK=1 to opt back in during an incident.
  • Updated the native deploy path so web production builds load the repo-root .env for build-time NEXT_PUBLIC_* values, then restore generated next-env.d.ts churn so the live checkout remains clean.
