Options Tape Smart-Flow Tinting Phase 03

Completed strict historical and server-row coverage by carrying smart_flow through option support hydration, loaded history rows, server durable option rows, and the canonical options-tape tint helper.

Outcome

/lookup/options-support now returns projected smart_flow explainability beside packets, smart-money, classifier hits, and NBBO. The frontend hydration scheduler caches those projections by packet ref and exposes them as smartFlowProjections.

OptionsTape now hydrates reusable visible tape rows, including rows returned from loadOlder history pages. Hydrated packets, NBBO, and smart-flow projections are merged into the same context builder used for live rows, so older loaded history rows can tint when packet or hypothesis evidence is discoverable.

Durable option row payloads now include typed support.smart_flow, and DurableTapeOptionRowsPane calls the same getOptionsTapeRowTint... helpers as OptionsTape instead of maintaining separate tint rules.

Proof

  • Historical row hydration: apps/web/features/options-tape/options-tape.test.ts asserts that createOptionsTapeSupportHydratingSource hydrates loadOlder rows, builds a support request for the older trace id, and maps the hydrated smart-flow projection into an options-tape-smart-flow-row tint.
  • Durable row tint parity: apps/web/features/options-tape/options-tape.test.ts compares getDurableOptionRowTint(row) to getOptionsTapeSmartFlowRowTint(projection) and expects identical metadata, class name, and style.
  • Durable server payload: services/api/tests/live.test.ts asserts durable option rows expose support.smart_flow.source_channel and support.smart_flow.refs.evidence_refs.
  • Bounded cache behavior: apps/web/features/terminal/hydration-scheduler.test.ts proves repeated support requests reuse a single lookup, return smartFlowProjections, and keep smart-flow misses bounded when packet support is already cached.

Reviewer Pass

Thermo-nuclear review found no blocking structural regression in the Phase 03 row-tinting shape: support hydration stays behind the scheduler cache, durable and reusable options rows share the canonical tint helper, and durable server payloads expose typed support.smart_flow instead of a catch-all object.

One scoped repair was made during review: lookupOptionsSupport now starts the independent NBBO lookup before waiting for packet support, while still delaying packet-dependent smart-money and classifier lookups until packet ids are known. A focused regression test proves the NBBO branch starts without waiting on packet support.
