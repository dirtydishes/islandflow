Execution Phases

Each phase is intentionally PR-sized. Run them serially because each phase creates the contract consumed by the next one.

  • Keep Options Tape row mapping local and behavior-preserving.
  • Make this the visual policy alerts will later consume.

Native Smart-Flow Runtime

  • Publish and persist native SmartFlowExplainabilityProjection rows.
  • Stop API smart-flow fetchers from projecting stored smart-money events.
  • Keep legacy emitters temporarily for comparison and rollback.

Derived Hypothesis Alerts

ghce.3
  • Add SmartFlowAlertEvent contracts and smart-flow-alerts surfaces.
  • Emit only for non-abstained projections.
  • Do not carry legacy score, severity, or classifier hits forward.

Alerts UI Migration

Consumer Cutover

ghce.5

Legacy Deletion And History Drop

ghce.6
  • Delete legacy emitters, routes, channels, storage helpers, UI state, and fallbacks.
  • Drop old derived history without backfill.
  • Preserve raw data, flow packets, smart-flow, and smart-flow-alert history.

Locked Decisions

These decisions remove ambiguity for implementers and reviewers.

Alerts Are Derived

  • No independent alert classifier.
  • No old score or severity fields.
  • Only non-abstained smart-flow projections alert.

History Is Dropped

  • No legacy derived backfill.
  • Old smart-money, classifier-hit, and legacy-alert history disappears at cutover.
  • Raw replay remains the source of recovery.

Reviewers Own CI

  • Reviewer threads inspect Forgejo CI.
  • They repair in-scope failures on the same branch.
  • No merge-ready callback while CI is pending or red.

Agent Topology

Use up to 20 subagents per phase. The count is a ceiling for broad phases, not busywork for narrow changes.

1 selector

Reads Beads, the phase doc, and reports the next unblocked task.

Up to 8 scouts

Read-only contract, compute, API, UI, storage, tests, docs, and deletion inventory.

Up to 4 worker helpers

Assist the implementation worker with bounded inspection and proposed patches.

Up to 6 review agents

Review focused slices with thermo-nuclear quality expectations.

1 lead reviewer

Consolidates findings, repairs safely, owns CI, updates Beads, and callbacks once.

1 orchestrator

Creates threads, receives callbacks, and closes out only after reviewer says CI is green.
